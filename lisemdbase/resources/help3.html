openLISEM database generator

This program generates a database for openLISEM by calling python scripts.

A temporary configuration file (lisemdbaseoptions.cfg) is generated automatically combining all options in the interface in your temp directory that is used by the python scripts.

Ini project files

The database also saves the options in a "ini" file, and ini files can be saved for each project in a folder of your choice. The list of ini files is checked when opening the dbase generator. Any missing ini files sare deleted from the list.

You can save the current interface configuration (save button), or save the current config under a new name (save as) and open the explorer to load a previous ini file. These are added to the dropdown list at the top.

CONDA environment

It is assumed that python, PCRaster, GDAL and other software is installed using Miniconda or Anaconda. In order to work, a conda environment must be created with the following commands:

  • conda create --name lisem python
  • conda activate lisem
  • conda install -c conda-forge pcraster owslib scipy gdal

If an environment is visible, it has PCRaster, GDAL and Python libraries and in principle the dbase generator can be executed.

LISEM Base and Map folders

In principle it is easiest to create the following folder structure for LISEM:

[your folder]/Base
[your folder]/Maps
[your folder]/Rain
[your folder]/res
[your folder]/run

These names are not compulsory. However, separate folders are needed for the Base forlder that stores the basic data (maps and tables) and the Maps folder that contains all input maps for LISEM. The rainfall data is stored in the "Rain" folder and the the LISEM run fikles in the run folder.

In the DBase generator you have to point to the "Base" and "Maps" folders.

The python script compiles all PCRaster and GDAL operations to create the LISEM input database.

The ESPG number is needed to ensure the right cartograohic projection in maps transformations.
