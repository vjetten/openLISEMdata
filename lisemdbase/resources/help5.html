Rainfall generation options

Two options are avilable: i. generate rainfall maps for the area from satellite images, and ii. generate an input file from daily data using synthetic rainfall.

i. Rainfall data from satellite images

Rainfall maps can be generated from the Global Precipitation Mission IMERG data. IMERG stands for Integrated Multi-satellitE Retrievals,based on a constellation of satellites. This is free data with rainfall intensity, for the entire globe in 11 km resolution, every 30 minutes. Data are available since 2000. The DBase generator does not download the data, but assumes that you have downloaded the necessary dataset in a folder.

They can be found here:

https://jsimpsonhttps.pps.eosdis.nasa.gov/imerg/gis/

The data is stored per month. You need the 30min tif files. Note that registraton is required.

The DBase generator asks for the folder where your dataset is stored. It assumes that all files for the entire run are stored in one folder (if you have multiple months, store them in one folder).

Specify the folder where the reprojected ranfall input data should be stored (usualy a subfolder under the rain folder in the project dbase  structure). The reprojection is done using the base dem and ESPG number.

LISEM needs an input text file that lists all images in the run with their time interval. LISEM uses Julian day number. So for instance noon (minute 720) on the first of June (day 152) is time 152:0720. The generated file looks like this:

# GPM data
2
time (ddd:mmmm)
filename
152:0000 3B-HHR-L.MS.MRG.3IMERG.20140601-S000000-E002959.0000.V06B.30min.map
152:0030 3B-HHR-L.MS.MRG.3IMERG.20140601-S003000-E005959.0030.V06B.30min.map
152:0060 3B-HHR-L.MS.MRG.3IMERG.20140601-S010000-E012959.0060.V06B.30min.map
etc.

The rainfall used in LISEM is from the interval that it is listed at, until the next time interval, when a new map is loaded.

There are 3 options:

  • conversion factor: the values for GPM are stored in 0.1 mm, so a factor of 10 is required to convert to mm.
  • time interval of the images: (30 minutes for GPM data)
  • the type of interpolation used: 0 = nearest neighbour; 1 = biliear interpolation; 2 = bicubic interpolation.

ii. Generate syntathic rainfall from daily data

Provide a text file with daily rainfall values for the area, with the format:

#header
 2
 day
Pday
 152 0.0
 153 1.23
 154 2.34
 etc.

It is assumed you have a relation between the daily rainfall amount and the maximum intensity (Imax) on that day. This can be obtained from stations with hourly data. For each day with rainfall a synthetic storm is generated that has 5 intervals:

  1. interval 1: intensity = 0.5*Imax
  2. interval 2: intensity = Imax
  3. interval 3: intensity = 0.5*Imax
  4. interval 4: intensity = 0.25*Imax
  5. interval 5: intensity = 0.125*Imax
  6. interval 6: intensity = 0.0625*Imax

Each inerval has a durtaion of approximately 1 hour, but adjusted to ensure that the sum of the timeintervals equals the daily total for that day.

The rainfall starts randomly during the day, between 3 a.m. and 3 p.m. This is arbitrary.
