openLISEM database generator

This program generates a database from a few baseline PCRaster and GTiff maps for the openLISEM model, using a series of python scripts.

A temporary configuration file (lisemdbaseoptions.cfg) is generated automatically by this interface, combining all options in your temp directory. This configuration fiel is used by the the python scripts to create the database.

Ini project files

You can save the current interface configuration (save button), or save the current config under a new name (save as) as an "ini" file, in a folder of your choice (usually the project folder). You can also load previous ini files. These are added to the dropdown list at the top.

The list of ini files is checked when opening the dbase generator. Any missing ini files sare deleted from the list.

CONDA environment

The python scripts use GDAL and PCRaster libraries. It is assumed that python, PCRaster, GDAL and other software is installed using Miniconda or Anaconda. In order to work, a conda environment must be created with the following commands:

  • conda create --name lisem python
  • conda activate lisem
  • conda install -c conda-forge pcraster owslib scipy gdal

If an environment exists, and it has a valid installation of PCRaster, GDAL and Python libraries it becomes visible in the dbase generator and should work.

LISEM Base and Map folders

In principle it is easiest to create the following folder structure for a LISEM database:

[your folder]/Base           => (folder with base data not directly used by LISEM, DEM, downloaded soil data, land use map etc)
[your folder]/Maps           => (the generated database for LISEM, this folder you use in the model itself)
[your folder]/Rain            => rainfall data
[your folder]/res              => model results form simulations
[your folder]/run              => run files for LISEM

These names are not compulsory but do not use the same folder for the base data and input maps. In the DBase generator you have to point to the "Base" and "Maps" folders.

The python scripts compiles all PCRaster and GDAL operations to create the LISEM input database. The main python script is "lisemDBASEgenerator.py" which should eb present in a folder claled 'scripts' that is created when you unzip this package. A series of python script for thhe different map ccategories are called by this main script.

The ESPG number is needed to ensure the right cartograohic projection in map transformations. Unfortunately, PCRaster maps do not store this information, so you have to procvide it.
